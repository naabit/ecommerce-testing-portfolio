
# Test Plan – E-commerce

## 1. Objetivo

Validar el correcto funcionamiento de los flujos principales del sistema e-commerce, asegurando una experiencia de usuario estable y libre de errores críticos.

## 2. Alcance

Se incluyen los siguientes módulos:

- Registro
- Login
- Catálogo
- Carrito
- Checkout

## 3. Fuera de alcance

- Testing de performance
- Seguridad avanzada
- Compatibilidad cross-browser (por ahora)

## 4. Tipos de pruebas

- Funcionales
- Exploratorias
- Pruebas negativas
- Smoke testing

## 5. Riesgos identificados

- Fallos en login que impidan acceso
- Errores en carrito que afecten compras
- Problemas en checkout (críticos para negocio)
- Manejo incorrecto de errores

## 6. Criterios de éxito

- Flujos principales completados sin errores críticos
- Errores correctamente reportados
- Evidencia documentada
